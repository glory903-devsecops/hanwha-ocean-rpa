# Dashboard Design & Style Guide (v14.1.0)

본 가이드는 **'운영 위선(Operational Priority)'** 중심의 대시보드로 개편된 v14.1.0 버전에 맞춰 업데이트되었습니다. 이제 대시보드는 정적인 차트 대신, 개별 작업장(Node)의 심각도에 따라 정렬된 **'우선순위 카드'** 시스템을 사용합니다.

---

## 🎨 디자인 시스템: `src/viz/theme.py`

모든 디자인 변수는 `theme.py` 내의 `THEME` 딕셔너리에 정의되어 있습니다.

### 1. 심각도별 색상 (Severity Colors)
`THEME["colors"]` 항목에 새로 추가된 심각도 대응 색상입니다.
- `critical`: **매우 위험** (빨간색 `#EF4444`) - 최상단 배치
- `warning`: **주의 필요** (호박색 `#F59E0B`) - 중간 배치
- `success`: **정상/최적** (에메랄드 `#10B981`) - 하단 배치

### 2. 레이아웃: 우선순위 카드 (Priority Cards)
기존의 복잡한 차트를 대체하여, 각 도크를 카드 형태로 시각화했습니다.
- **좌측 보더(Left Border)**: 심각도 색상이 두꺼운 선으로 표시되어 즉각적인 인지가 가능합니다.
- **애니메이션**: '위험(Critical)' 등급의 카드는 은은한 펄스(Pulse) 효과를 통해 주목도를 높였습니다.

---

## 🛠️ 대시보드 수정 워크플로우

1. **심각도 정렬 로직 수정**: `src/viz/dashboard.py` 내의 `get_severity_score` 함수를 수정하여 어떤 키워드가 '위험'으로 분류될지 정의할 수 있습니다.
2. **카드 스타일링**: `dashboard.py`의 HTML 템플릿 내 `cards_html` 생성 루프에서 여백(Padding), 둥글기(Radius) 등을 조정합니다.
3. **대시보드 재생성**: 수정을 마친 후 아래 명령어를 실행합니다.
   ```powershell
   python src/viz/dashboard.py
   ```

## ✨ 시각적 체크리스트 (Design QA)
- [ ] **스크롤 가독성**: 상하로 스크롤 할 때 각 카드의 경계가 명확하고 텍스트 레이아웃이 깨지지 않는가?
- [ ] **상단 KPI 바**: 전체 공정률과 가동 중인 알럿(Alert) 개수가 상단 헤더에서 명확히 보이는가?
- [ ] **색상 대비**: 다크 모드 배경에서 위험(Red)과 주의(Amber) 색상이 충분히 돋보이는가?

> [!IMPORTANT]
> 본 대시보드는 실시간 운영 의사결정을 위해 **'위험도가 높은 노드'를 강제로 최상단에 고정**합니다. 시각적 스타일은 `theme.py`에서, 정렬 우선순위는 `dashboard.py`에서 관리하십시오.
